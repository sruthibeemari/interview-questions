Git and Github Questions

Git is a version control system that tracks changes in computer files. It is used to push the code from our system.

Git is an open-source version control tool that tracks changes to files, while GitHub is a cloud-based platform that allows users to share and collaborate on code and helps to do projects together.

The (git init) command is used to initialize Git in your local directory.

  • Working directory
  • Staging area
  • Local repository
  • Remote repository

(git add .) is used to add all files and (git add filename) is used to add a particular file.

(git commit -m "message")

(git remote add origin) `

A repository is a place where you can store your code, your files, and each file's revision history.

(git push origin main)

(git branch -M branchname)

(git branch)

(git checkout branch name)

(git log --oneline)

The (git status) command shows the current state of your Git repository, including the status of your working directory and staging area.

(git fetch), (git pull), (git clone)

A fork in GitHub is a copy of a repository that shares the same code and visibility settings as the original repository.
  1. Find the repository: Go to the repository page that you want to fork.
  2. Fork the repository:
    • In the top-right corner of the repository's page, you'll see a "Fork" button.
    • Click on the "Fork" button. GitHub will create a copy of the repository under your account.

  • git fetch: Downloads commits, files, and references from a remote repository, but doesn't merge or modify your working directory.
  • git pull: Combines `git fetch` and `git merge`—it fetches the latest changes and directly updates your local branch by merging them.

  1. git stash:
    • Temporarily saves (stashes) your uncommitted changes without committing them, allowing you to work on something else.
    • You can later reapply these changes using (git stash pop) or (git stash apply).
  2. git diff:
    • Shows the differences between your working directory and the last commit (or between any two commits).
    • Useful for seeing changes made to files before committing them.

  1. git remote:
    • Manages connections to remote repositories (e.g., GitHub).
    • You can use it to add, remove, or list remotes.
  2. git reset:
    • Moves the HEAD pointer and optionally modifies the working directory and staging area.
    • Useful for undoing commits or unstaging changes.
